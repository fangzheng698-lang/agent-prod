# Security Policy

## Supported Versions

Security fixes are currently handled for the latest `main` branch and the
latest published release.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security reports.

Report vulnerabilities privately by contacting the repository owner through
GitHub. Include:

- affected version or commit SHA
- impact summary
- reproduction steps or proof of concept
- relevant logs, traces, or configuration
- suggested mitigation, if known

We aim to acknowledge valid reports within 7 days.

## Sensitive Data

Do not include API keys, tokens, customer data, private prompts, internal URLs,
or production traces in public issues or pull requests. Redact sensitive values
before sharing logs.
